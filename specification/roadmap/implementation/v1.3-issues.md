## v1.3 — ASR input (full voice loop)

### PYR-011 — Cloud ASR client (Ukrainian PCM16 → transcript)

**Description:**
Add a speech-to-text client: send the captured push-to-talk audio (16 kHz mono PCM16 in `g_pcm`) to a cloud Ukrainian ASR over HTTPS and get back a transcript. To keep the ESP32 simple, prefer an API that accepts **raw PCM16** (no multipart/WAV assembly on-device), mirroring the v1.2 TTS pattern.

**Provider (decided): Deepgram** prerecorded — `POST https://api.deepgram.com/v1/listen?model=nova-2&language=uk&encoding=linear16&sample_rate=16000` with `Authorization: Token <key>`, body = the raw `g_pcm` bytes (no WAV header, no multipart — mirrors the v1.2 TTS raw-bytes POST). Returns JSON; transcript at `results.channels[0].alternatives[0].transcript` (+ `confidence`). (Whisper via OpenAI/Groq — multipart + on-device WAV — stays a documented fallback only.)

**What needs to be done:**
- Add an ASR config block to `config.h`: `ASR_ENDPOINT`, `ASR_API_KEY`, `ASR_MODEL`, `ASR_LANG` (`uk`), `ASR_SAMPLE_RATE` = `AUDIO_SAMPLE_RATE`.
- Implement `asrTranscribe(pcm, samples, &transcript, &err)`: HTTPS POST the raw PCM16 (reuse `WiFiClientSecure` + `HTTPClient`; `Authorization: Token` + `Content-Type: application/octet-stream`), read the JSON response.
- Keep response parsing in a **pure, host-testable** helper — an `asr_api.h` mirroring `chat_api.h`: `parseAsrTranscript(body, &transcript, &confidence, &err)` extracting `results.channels[0].alternatives[0].transcript`; map API-error/empty to readable errors.

**Dependencies:** None (builds on the existing HTTPS stack and the v1.1 capture buffer).

**Expected result:**
Given recorded PCM16, the client returns a Ukrainian transcript (or a clear error).

**Acceptance criteria:**
- [ ] Direct HTTPS (TLS) call sends raw PCM16 and returns a transcript for spoken Ukrainian
- [ ] Endpoint/key/model/lang loaded from `config.h`
- [ ] Response parsing extracts the transcript (+ confidence); API-error/empty → readable error
- [ ] Parsing is unit-tested against a recorded/mock ASR response (`pio test -e native`)
- [ ] On-device: a spoken phrase yields a transcript logged to serial (manual check)

### PYR-012 — Full voice loop (button → ASR → LLM → TTS → speaker)

**Description:**
Rewire the push-to-talk button from the v1.1 record→playback *loopback* to the real exchange: hold to record, release to **transcribe → LLM → TTS → speak**. The ASR transcript becomes the `text_in` for the existing LLM turn, and the reply flows through the v1.2 TTS path — reusing history, retry, and the mic↔speaker switch.

**What needs to be done:**
- On button release: take the captured `g_pcm` (v1.1), call `asrTranscribe` (PYR-011) → transcript.
- Feed the transcript into the existing `llmTurn` (same persona/history as the serial path) → reply; then the v1.2 TTS → playback.
- Mirror the recognized `text_in` (transcript) and the `reply` to serial for debugging the whole chain.
- Drive LCD states through the turn: `listening → thinking → speaking → idle`.
- Keep the serial `text_in` path working too (typed lines still chat).

**Dependencies:** PYR-011 (and the done v1.1 capture + v1.2 TTS/LLM chain)

**Expected result:**
You hold the button, speak Ukrainian, release, and hear the assistant's spoken Ukrainian reply — end to end on the device.

**Acceptance criteria:**
- [ ] Hold→speak→release produces: transcript → LLM reply → spoken reply, end to end
- [ ] The transcript and reply are mirrored to serial
- [ ] LCD reflects listening / thinking / speaking / idle
- [ ] Reuses the existing LLM turn (history/retry) and v1.2 TTS+playback (no duplicate paths)
- [ ] The serial text path still works; verified on hardware

### PYR-013 — ASR robustness (empty/failed recognition, timeout, noise gate)

**Description:**
Make the voice loop hold up: don't send silence/noise to the cloud or feed an empty/garbage transcript to the LLM. On empty or failed recognition, re-prompt instead of answering noise; bound the ASR call with a timeout.

**What needs to be done:**
- **Noise/length gate** before ASR: using the v1.1 capture stats (peak/level + duration), skip ASR when the recording is too short or below a level threshold (don't POST silence); log it.
- **Empty/low-confidence transcript:** if the transcript is empty (or confidence below a threshold), **re-prompt** — a short spoken-or-logged "Не почула, повтори, будь ласка" — rather than calling the LLM.
- **ASR timeout** (bounded per-stage budget) + graceful error; the loop never hangs and returns to `idle`.
- Keep the gate/decision logic in pure helpers where feasible (e.g. `shouldTranscribe(samples, peak, minMs, minPeak)`), host-tested.

**Dependencies:** PYR-012

**Expected result:**
Silence, noise, and failed recognition never reach the LLM; the device re-prompts cleanly and the loop never hangs.

**Acceptance criteria:**
- [ ] Too-short / too-quiet recordings are gated out before ASR (no API call), with a log
- [ ] Empty/low-confidence transcript → re-prompt, not an LLM call on noise
- [ ] ASR timeout/error handled with a clear message; loop returns to idle, never hangs
- [ ] Gate/decision logic unit-tested (`pio test -e native`); a reliability pass over typical scenarios succeeds on hardware

## v1.3 scope notes

**Total effort:** ~1 week for a single developer (a short chain; PYR-011 is the bulk).

**Critical path:** PYR-011 → PYR-012 → PYR-013. The DoD (speak → hear a Ukrainian reply) is met at PYR-012; PYR-013 makes it robust.

**Phase DoD (ROADMAP §v1.3):** a full spoken exchange in Ukrainian, end to end, from the device.

**Provider (decided): Deepgram** prerecorded with `encoding=linear16` — raw 16 kHz PCM16 POST, no multipart/WAV on the ESP32, Ukrainian via `nova-2`. The open detail is the exact model (`nova-2` vs `nova-3`) and the key. Whisper (OpenAI/Groq) stays a documented fallback only. The recorded clip is bounded by the v1.1 `REC_MAX_MS` buffer (~5 s), which suits push-to-talk utterances.

**Reused infrastructure:** the HTTPS/TLS client + config pattern (v0.2), the push-to-talk capture into `g_pcm` (v1.1), and the LLM turn + TTS/playback chain (v0.3 + v1.2). v1.3 adds only the ASR client and the input wiring — no new audio path.

**End-of-utterance:** v1.3 uses **button release** as end-of-utterance (push-to-talk). Pause-based VAD (silence detection bounded by `recog_patience`) is a **v1.4** concern, not here.

**Secrets:** the ASR key joins the Wi-Fi / LLM / TTS keys in the gitignored `config.h`; only `config.example.h` is committed.
