Pyramid -- Release Notes
========================

Version 1.3.0 (2026-06-03)
--------------------------
v1.3 milestone -- Voice, phase 3: ASR + the full voice loop. Hold the button,
speak Ukrainian, and hear Piramida's spoken reply: mic -> Deepgram ASR ->
Claude -> ElevenLabs TTS -> Echo Base speaker, end to end. Direct-to-cloud;
serial remains a debug channel. v1.4 (states/UX) remains.

- PYR-011 Cloud ASR client: Deepgram prerecorded transcription over HTTPS,
  captured PCM16 -> Ukrainian transcript (model nova-2, language uk). Pure,
  host-tested response parser (asr_api.h).
- PYR-012 Full voice loop: push-to-talk records the mic, transcribes it, runs
  the shared LLM->TTS turn, and plays the reply -- the complete button -> ASR
  -> LLM -> TTS -> speaker path on the device.
- PYR-013 ASR robustness + latency: noise gate (min duration/peak) skips silence
  and accidental taps; low-confidence transcripts re-prompt instead of answering
  noise; bounded retry on transient ASR errors. In-place G.711 mu-law encoding
  halves the upload (fixes Deepgram 408 SLOW_UPLOAD); turbo TTS model trims
  latency. Pure, host-tested mu-law encoder (ulaw.h) + audio gate (audio.h).
- Default persona = Piramida for v1 (terse Ukrainian helper). The richer Lili
  Name+Canon stays in the spec for the v2 server-side Role.

Version 1.2.0 (2026-06-03)
--------------------------
v1.2 milestone -- Voice, phase 2: TTS output (spoken replies). Type over USB
serial and hear Claude's Ukrainian reply spoken through the Atomic Echo Base.
v1 still in progress -- v1.3 (ASR) and v1.4 (states/UX) remain.

- PYR-008 Cloud TTS client: ElevenLabs text-to-speech over HTTPS,
  output_format=pcm_16000 (raw 16 kHz mono PCM16, no on-device decode), Ukrainian
  via eleven_multilingual_v2. Pure, host-tested request builder.
- PYR-009 TTS -> playback pipeline: after the LLM reply, the text is sent to TTS
  and played through the Echo Base speaker using the v1.1 playback path; the
  reply text stays on serial for debugging.
- PYR-010 TTS robustness: UTF-8 boundary-safe length cap (TTS_MAX_CHARS), bounded
  timeout, and a spoken-or-logged fallback so a TTS hiccup never hangs the loop.
- Buffered playback with a terse persona keeps replies smooth and complete
  (verified on hardware). Gapless streaming of long replies is deferred to v1.4
  (needs a background audio task).

Version 1.1.0 (2026-06-03)
--------------------------
v1.1 milestone -- Voice, phase 1: Audio I/O and the PlatformIO migration.
(Version notation A.B.C: A = global version (v1), B = phase (1), C = post-
release fix.) v1 is still in progress -- v1.2 (TTS), v1.3 (ASR), and v1.4
(states/UX) remain.

- PYR-004 PlatformIO migration: firmware moved off the Arduino IDE to
  PlatformIO (src/ layout, atoms3r + native envs); identical build/flash.
- PYR-005 Native test environment: pio test -e native (Unity) runs the pure-
  logic suites (line reader, text_in, history, backoff, SSE, audio) on the host.
- PYR-006 I2S audio capture: push-to-talk record of 16 kHz mono PCM16 from the
  Atomic Echo Base mic (ES8311), with level/clip checks and a max duration.
- PYR-007 I2S audio playback: play the captured buffer on release -- the
  record -> playback loop. Verified on hardware (audible, stable).

Version 0.3.1 (2026-06-03)
--------------------------
Streaming refinement of the v0 text chat. The device now streams Claude's
reply token by token and reports per-turn latency and token usage.

- Streaming LLM replies (SSE): the request sets stream:true and the reply is
  printed to serial token by token as it arrives, via a pure, host-tested
  decode pipeline (sse.h: HTTP chunked-transfer decode -> data: line
  extraction -> event parsing). The on-device TLS socket read is covered by
  compile + the manual on-hardware check.
- Per-turn stats line after each reply: first_token (genuine
  time-to-first-token), total (whole turn incl. any retries), and token
  counts (in / out / total) from the API usage.

Version 0.3.0 (2026-06-03)
--------------------------
v0 (Text chat over serial) is COMPLETE -- a typed line over USB serial gets a
real LLM reply in Ukrainian, with short context and graceful failure handling.
(Minor-bump-per-phase cadence; this marks v0 done.)

- PYR-003 (v0.3) Quality and UX: a short rolling in-RAM conversation history
  (HISTORY_MAX_TURNS, windowed to start with a user turn) is resent with each
  request for context; the LLM call does a bounded retry with capped
  exponential backoff on transient failures (transport / 429 / 5xx) and
  surfaces a clear error line otherwise -- the loop never hangs; Wi-Fi loss
  auto-recovers via non-blocking exponential backoff with input paused while
  offline; the LCD shows idle / thinking / error / offline. Only successful
  turns are committed to history. New pure, host-tested headers history.h and
  backoff.h plus a retry classifier.
- LLM switched to Anthropic: Claude via the Messages API (default
  claude-haiku-4-5-20251001, config-selectable), replacing the
  OpenAI-compatible client.

Version 0.2.0 (2026-06-03)
--------------------------
Phase v0.2 milestone (minor-bump-per-phase cadence, not the v1-complete
mapping). v0 (Text chat over serial) is still PARTIAL -- PYR-003 (quality &
UX, v0.3) remains open.

- PYR-002 (v0.2) Text chat loop: each serial text_in is sent to a cloud LLM
  over direct HTTPS (persona / model / endpoint / key from config.h) and the
  model's reply is printed back. The turn is synchronous, with a "thinking..."
  log while waiting. Request building and reply parsing use ArduinoJson v7 in
  a pure, host-tested module (7 checks vs recorded mock responses); HTTP/JSON
  errors surface as a readable line without hanging the loop. Replies are in
  Ukrainian per the persona prompt.
- Docs: added a root README (project overview, v0-v3 progression) and a
  VERSION file.

Version 0.1.0 (2026-06-03)
--------------------------
First tagged milestone. v0 (Text chat over serial) is PARTIAL -- only phase
v0.1 is implemented. PYR-002 (text chat loop, v0.2) and PYR-003 (quality &
UX, v0.3) remain open; see
specification/roadmap/implementation/v0-execution-report.md.

- PYR-001 (v0.1) Device skeleton and serial: AtomS3R + Echo Base bring-up
  with M5Unified (display, button, LCD status line); Wi-Fi STA from a
  gitignored config header; USB-CDC serial at 115200 turned into a
  line-oriented text channel -- a typed line maps to a text_in event and is
  echoed back. All status logging routes through logf() and is gated by
  DEBUG_SERIAL.
- Pure, Arduino-free line reader and text_in parser, with a host unit test
  (10 checks passing); firmware validated by arduino-cli compile for
  m5stack:esp32:m5stack_atoms3r (flash 32%, RAM 14%).
- Repo baseline: specification (MISSION / ARCHITECTURE / ROADMAP), CLAUDE.md,
  and the spec -> issues -> execute -> release workflow skills.
